What is the Plain Engine?

          The Plain Engine is a combined math and physics engine that will be used for real time simulations. It is designed to be easily understandable with simple algorithms, while also implementing high performance techniques that make it worth using. The purpose of the engine and this website is to provide a learning experience for myself and for others if they want to create their own engines, as most resources such as physics or rendering textbooks are either expensive or hard to come across. It is for this reason that I plan to provide a list of all the sources I used for each section of the engine, to show where I got the algorithms from and so you can find more in-depth explanations to learn from.

What Platforms does the Plain Engine Support?

The Plain Engine can ran on the following systems:

  • Windows.
    • 32 Bit.
    • 64 Bit.
  • Linux.
  • MacOS.
  • Android.
  • Unix.
    • Unix is the default failsafe.

What is the Plain Engine Dependent on?

          The Plain Engine is dependent on Assimp, Clang, CMake, GLFW, Glslang, SPIR-V Tools, and Vulkan API. These dependancies do not relate to the math or physics engine directly, as they are made entirely from scratch. The dependancies listed are required for platform control, file reading, rendering APIs, and compiling purposes only.

Dependency: Description: Webpage:
Assimp A multi-platform 3D model importer and exporter. Assimp Webpage
Clang A cross-platform C++ compiler built on LLVM. Clang Webpage
CMake A build-system generator. CMake Webpage
GLFW A cross-platform windowing and input library. GLFW Webpage
Glslang A GLSL/HLSL to SPIR-V shader compiler. Glslang Webpage
SPIR-V Tools Utilities for SPIR-V shader inspection and validation. SPIR-V Tools Webpage
